# Concept: 文脈体験プラットフォーム

## 目的

このプロダクトの本質は、株・会社・新人教育・創作など、特定領域に閉じたアプリケーションではない。

大量で複雑な情報を、ユーザーが自分の手に収まるサイズへ少しずつ削ぎ落とし、**「自分で見えるようになった」「自分で掴み取った」**と感じられる体験を作るための基盤である。

本ドキュメントでは、この基盤を仮に **文脈体験プラットフォーム** と呼ぶ。

---

## 中心思想

ユーザーは、最初から大量の情報を処理できない。

特に株式投資、業務理解、新人教育、創作設計のような領域では、単語・イベント・時系列・関係者・背景・暗黙知が複雑に重なっている。

そのため、単に情報を集めたり、AIに要約させたりするだけでは不十分である。

必要なのは、以下の流れを自然に体験させること。

```txt
身近なものから始める
↓
少し外側にある「わかりそうでわからないもの」を見つける
↓
近づいた時だけ説明する
↓
次に見る方向を示す
↓
チェックポイントごとに理解をまとめる
↓
最初とは景色が変わったことを実感させる
```

これは「教える」よりも **発見を演出する** 設計である。

## 表のテーマと裏のテーマ

この基盤は、常に表と裏の二重構造を持つ。

### 表の大きなテーマ

ユーザーが最初に気になること。

株であれば、

```txt
この株は上がるのか
下がるのか
持っていて大丈夫か
このニュースは危ないのか
この投資信託はなぜ動いたのか
```

新人教育であれば、

```txt
この対応で合っているのか
誰に確認すればいいのか
どこでミスりやすいのか
何を優先すべきなのか
```

表のテーマは、ユーザーの不安・興味・欲望に近い。

### 裏の中くらいのテーマ

用語・イベント・工程から、その領域の成り立ちを理解すること。

株であれば、

```txt
自己株式取得
新株予約権
下方修正
決算短信
大量保有報告書
信託報酬
為替ヘッジ
リバランス
```

新人教育であれば、

```txt
一次対応
エスカレーション
承認
納期回答
請求
契約
確認中
ナレッジ化
```

ここでは、用語の意味だけでなく、イベントの現在地を扱う。

### 裏の大きなテーマ

あらゆる情報は、別の情報へ接続している。

株なら、金利・為替・政策・技術・戦争・人口・消費・文化・業界動向が、企業価値や市場評価に接続する。

業務なら、顧客・社内ルール・部署間関係・契約・過去事例・責任範囲が、目の前の判断に接続する。

この基盤は、単発の情報ではなく、情報同士のつながり方を体験させる。

---

## 入口は軽く、裏側は深く

ユーザーにいきなり全体像を見せてはいけない。

初心者に対して、16年分の講義ノートを渡すような体験は避ける。

最初に出すべきものは、軽い入口である。

```txt
今日の1枚
この単語なに？
今どの段階？
ここだけ見ればいい
似た例もあります
```

裏側には、深い構造がある。

```txt
用語図鑑
イベントマップ
時系列
複数対象比較
仮説メモ
行動ログ
振り返り
```

しかし、ユーザーが近づくまで説明しすぎない。

---

## 1枚型

1枚型は、ユーザーが最初に触れるカードである。

役割は、大量の情報から入口を1つだけ切り出すこと。

株の例：

```txt
今日の1枚：
自己株式の取得状況に関するお知らせ

これは何？
すでに決まっていた自社株買いが、今どれくらい進んだかを報告する資料です。

今日見るところ：
- 今月いくら買ったか
- 累計でいくら買ったか
- 上限に対して何％進んだか

今日の理解：
自己株式取得は「決定」「状況」「終了」で意味が違う。
```

新人教育の例：

```txt
今日の1枚：
顧客から「最短納期でお願いできますか？」と連絡が来た

これは何？
すぐに確約せず、在庫・権限・影響範囲を確認する場面です。

今日見るところ：
- 在庫はあるか
- 納期回答の権限は誰にあるか
- 確約してよい段階か

今日の理解：
「確認します」は返答ではなく、次の行動の約束。
```

## 図鑑型

図鑑型は、ユーザーが見つけた用語・イベント・概念を蓄積する場所である。

ソシャゲの図鑑に近いが、目的は課金圧ではなく、発見と理解の蓄積である。

```txt
新しい用語を発見しました
図鑑に追加されました
似たイベントがあります
過去の事例があります
```

図鑑は、単なる定義集ではない。

```txt
ざっくり意味
なぜ大事か
まず見るポイント
よくある勘違い
関連イベント
過去事例
```

を持つ。

---

## イベントマップ

イベントマップは、用語の意味ではなく、出来事の流れと現在地を表す。

例：自己株式取得

```txt
[取得決定] → [取得状況] → [取得終了] → [消却]
                 ↑ 今ここ
```

初心者が知りたいのは、しばしば「言葉の意味」ではない。

```txt
これは始まった話なのか
途中経過なのか
終わった話なのか
前からあった話なのか
次に何が起こり得るのか
```

である。

したがって、用語図鑑とイベントマップは分ける。

---

## 同心円と地図

ユーザー導線は同心円型がよい。

```txt
中心：
  自分の銘柄・投信・案件・関心

少し外：
  今日の1枚

さらに外：
  用語・イベント

もっと外：
  過去事例・同業比較・業界文脈
```

一方で、知識構造そのものは普通の地図に近い。

```txt
大陸
飛び地
分岐路
一本道
隣接領域
```

自己株式取得のように一本道に近いものもあれば、資金調達のように複数の領域へ分岐するものもある。

## 投資信託と個別株の違い

株領域において、個別株と投資信託は同じUIで扱えるが、カード内容と図鑑のテイストは分けるべきである。

### 個別株

個別株は「選手カード」に近い。

見る対象は、会社に何が起きたか。

```txt
決算
資金調達
受注
配当
自己株式取得
ガバナンス
大株主
```

### 投資信託

投資信託は「チーム・リーグ・戦術カード」に近い。

見る対象は、商品構造と外部環境である。

```txt
指数
ベンチマーク
組入上位銘柄
地域比率
セクター比率
信託報酬
為替
金利
リバランス
```

個別株は出来事カード。  
投資信託は世界観カード。

## ユーザー行動の考え方

最初から自由メモを書かせない。

「分からないことをメモしてください」は、初心者にとって重すぎる。

最初は選択式でよい。

```txt
分かった
なんとなく分かった
まだ分からん
あとで見る
重要そう
```

ユーザーがよちよち歩き出したら、チェックポイントごとにまとめる。

さらに進んだユーザーだけが、自由メモ・仮説メモ・比較ビューへ進む。

---

## 行動心理上の原則

この基盤は、以下の行動心理を前提にする。

```txt
小さな成功体験
好奇心の隙間
選択肢の削減
自分ごと化
進捗の可視化
恐怖ではなく理解による誘導
```

「勉強しろ」と言わない。  
「気づいたら文脈を読んでいた」にする。

## AI分析の位置づけ

行動ログや回答データをマスキングしたうえでAIに分析させることは有効である。

ただし、AI分析は評価ではなく、観察コメントとして扱う。

```txt
一般論：
  このユーザーは補足情報を確認しながら進める傾向があります。

辛口：
  慎重なのは良いが、確認項目を増やしすぎて結論を出す筋力が育っていない可能性があります。
```

人事評価や能力断定には使わない。

特に新人教育領域では、以下を避ける。

```txt
採用合否
人事評価
ランク付け
能力断定
```

目的は、本人と教育担当が「どこで詰まりやすいか」を見ることにある。

---

## この基盤が扱うもの

この基盤は、以下を扱う。

```txt
1枚カード
用語図鑑
イベントマップ
タイムライン
比較ビュー
チェックポイント
未解明ワード
行動ログ
振り返り
```

ドメイン知識そのものはcoreに入れない。

株、新人教育、創作、業務フロー、法律、医療などの知識はpluginとして扱う。

---

## 一文で言うなら

この基盤は、巨大で暗黙知だらけの情報を、ユーザーが自分の手に収まるサイズへ削ぎ落とし、  
**「自分で見えるようになった」と感じるための文脈体験プラットフォーム**である。
